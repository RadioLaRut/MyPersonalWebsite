## 配置首页作品列表

首页的作品列表在 `src/app/page.tsx` 文件中配置。

### 修改首页作品

打开 `src/app/page.tsx`，找到 `projects` 数组：

```typescript
const projects = [
  {
    title: "INSIGHT",
    subtitle: "First-Person Puzzle / Unity",
    imageSrc: "/images/Insight/InsightCover.png",
    link: "/works/insight",
  },
  // 添加更多作品...
];
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `title` | string | 作品名称（显示为大标题） |
| `subtitle` | string | 作品副标题/分类 |
| `imageSrc` | string | 封面图片路径（相对于 public 目录） |
| `link` | string | 跳转链接（指向作品详情页） |

## 配置作品详情页

作品详情页的数据配置位于 `src/app/works/[id]/page.tsx` 文件中的 `projectsData` 对象。

### 作品数据结构

每个作品包含以下字段：

```typescript
{
  title: "作品标题",
  idNum: "01",                    // 作品编号
  heroImage: "/images/xxx.png",   // 英雄区大图
  description: "作品描述文案...", // 主要介绍文字
  col1: {
    title: "第一栏标题",
    text: "第一栏描述",
    img: "/images/xxx.png"
  },
  col2: {
    title: "第二栏标题",
    text: "第二栏描述",
    img: "/images/xxx.png"
  },
  col3: {
    title: "第三栏标题",
    text: "第三栏描述",
    img: "/images/xxx.png"
  },
  gallery: [                      // 图库数组
    "/images/gallery/01.png",
    "/images/gallery/02.png"
  ],
  navLink: "https://...",         // 外部链接（可选，如 BiliBili 视频）
  nextId: "下一个作品ID",
  nextName: "下一个作品名称",
  nextBg: "/images/xxx.png"       // 下一个作品的背景图
}
```

### 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `title` | 是 | 作品标题 |
| `idNum` | 是 | 作品序号（用于显示 Project 01） |
| `heroImage` | 是 | 首屏英雄区图片路径 |
| `description` | 是 | 作品的主要描述文案 |
| `col1` / `col2` / `col3` | 是 | 三个技术/细节展示栏 |
| `gallery` | 是 | 作品图片展示阵列 |
| `navLink` | 否 | 外部链接（如演示视频） |
| `nextId` | 是 | 下一个作品的 ID（用于"下一项目"导航） |
| `nextName` | 是 | 下一个作品的显示名称 |
| `nextBg` | 是 | 下一个作品的预览图 |

## 灯光作品集配置及对比组件使用

对于“灯光作品集 (Lighting Portfolio)”以及存在**有光照/无光照**对比情况的项目，我们需要使用专用的对比功能。

### 1. 二级图集索引页配置
灯光作品集的首个落地页是一个展示多个“集 (Collections)”的网格。采用极其克制的数字序号和幽暗文本来指引，并使用不对称的网格（基于 12列全局网格）。配置项通常为一个存放了多个 collection 对象的数组，每个对象包含图片封面和对应的子路由。

### 2. Breakdown 详情页的光照对比 (Before/After Slider)
在单个作品的详情页中，我们将摒弃花哨的特效（如手电筒），转而采用“**纯粹且专业**”的图像滑动对比控件（Image Slider Component）。
如果某张作品有 Unlit 版本，数据结构会扩展支持 `unlitImg` 字段：

```typescript
gallery: [
  {
    litImg: "/images/Lighting/scene_01_lit.png",       // 必填：有光照图
    unlitImg: "/images/Lighting/scene_01_unlit.png",   // 可选：无光照对比底图
    caption: "场景 01 - 主光照设定"
  },
  // ... 其他图片
]
```
渲染时，若检测到 `unlitImg` 存在，系统将自动挂载滑块组件，用户拖动细线即可直观地横向擦除/展示光照效果。

## 添加新作品

### 步骤 1: 在详情页数据中添加条目

在 `src/app/works/[id]/page.tsx` 的 `projectsData` 对象中添加新条目：

```typescript
"my-new-project": {
  title: "我的新作品",
  idNum: "09",
  heroImage: "/images/MyProject/hero.png",
  description: "这是我的新作品描述...",
  col1: { title: "角色设计", text: "描述文字", img: "/images/MyProject/01.png" },
  col2: { title: "程序实现", text: "描述文字", img: "/images/MyProject/02.png" },
  col3: { title: "美术风格", text: "描述文字", img: "/images/MyProject/03.png" },
  gallery: [
    "/images/MyProject/gallery1.png",
    "/images/MyProject/gallery2.png"
  ],
  navLink: "https://example.com/video",
  nextId: "another-project",
  nextName: "另一个作品",
  nextBg: "/images/AnotherProject/cover.png"
}
```

### 步骤 2: 在首页添加入口（可选）

在 `src/app/page.tsx` 的 `projects` 数组中添加：

```typescript
{
  title: "我的新作品",
  subtitle: "游戏类型 / 引擎",
  imageSrc: "/images/MyProject/cover.png",
  link: "/works/my-new-project",
}
```

### 步骤 3: 在作品列表页添加入口（可选）

在 `src/app/works/page.tsx` 的 `works` 数组中添加：

```typescript
{ id: "my-new-project", title: "我的新作品", category: "游戏类型", imageSrc: "/images/MyProject/cover.png", desc: "简短描述" }
```

## 图片资源管理

### 图片存放位置

所有图片资源放在 `public/images/` 目录下：

```
public/
└── images/
    ├── Insight/           # 按作品分类的文件夹
    ├── STV/
    ├── Prometheus/
    ├── Others/
    ├── TrainStation/
    └── 封面和结尾/
```

### 添加新图片

1. 将图片放入 `public/images/` 下的对应文件夹
2. 在配置中引用路径（相对于 public 目录，以 `/images/` 开头）

### 图片路径示例

```typescript
// public/images/Insight/cover.png
heroImage: "/images/Insight/cover.png"

// public/images/abc.jpg
imageSrc: "/images/abc.jpg"
```

## 页面路由说明

| 路由 | 页面 | 配置文件 |
|------|------|----------|
| `/` | 首页 | `src/app/page.tsx` |
| `/works` | 作品列表页 | `src/app/works/page.tsx` |
| `/works/:id` | 作品详情页 | `src/app/works/[id]/page.tsx` |
| `/contact` | 联系页 | `src/app/contact/page.tsx` |

### 路由参数

作品详情页使用动态路由 `[id]`：
- 访问 `/works/insight` 显示 INSIGHT 作品
- 访问 `/works/prometheus` 显示 PROMETHEUS 作品

## Puck CMS 可视化编辑器使用

为了方便你直接在浏览器中像拼图一样排版并立刻生效到最终前端网页中，项目中内置了高安全性的 Puck 可视化编辑器系统。它会将你编辑的内容自动保存为 JSON 文件存储到 `content/pages` 目录下。

### 1. 如何进入编辑模式

你只需要记住两条启动命令，测试模式不再需要手动敲环境变量：

1. 在终端中运行以下命令启动特权开发服务器：
   ```bash
   # 普通模式
   npm run dev

   # 测试模式（显示 playground、允许选中、可访问 /admin）
   npm run dev:test
   ```
2. 在浏览器中访问管理地址：
   **[http://localhost:3000/admin](http://localhost:3000/admin)**

> **注意：** 尝试吧如果你仅仅使用 `npm run dev` 启动，访问 `/admin` 只会显示白屏和 `404 Not Found` 错误，以此保证系统在未授权时不存在任何内容泄露。

### 2. 编辑模式基础操作

- **布局操作**：左侧边栏为可用积木（Components），包含 Hero 标题、图文段落等。你可以拖拽这些积木到中央的大画布中。
- **编辑内容**：选中任意放入画布的组件，右侧栏即可对其内容进行实时修改、填写文字、上传选择图片配置。
- **本地持久保存**：点击页面右上角的 `Publish` 按钮（或发布按钮）。系统会自动将当前页面配置内容以原子替换的方式存储到代码库根目录下的 `content/pages` 文件内。

### 3. 多页面跳转编辑

`/admin` 默认编辑的是 `index` 首页的内容档案。如果你想排版其它具体子页面（例如 `/p/lighting-portfolio`），直接在管理地址后跟入对应路径：

- 访问 **[http://localhost:3000/admin/lighting-portfolio](http://localhost:3000/admin/lighting-portfolio)**
- 编辑好之后可以独立发布。

前台可以通过访问 `[http://localhost:3000/p/lighting-portfolio](http://localhost:3000/p/lighting-portfolio)` 来实时预览这个静态发布的页面。

## 开发命令

```bash
# 安装依赖
npm install

# 启动开发服务器（常规模式）
npm run dev

# 启动并开启 Puck 本地可视化编辑器（随后可访问 http://localhost:3000/admin）
npm run dev:test

# 构建生产版本
npm run build

# 启动生产服务器
npm start
```

## 注意事项

1. **路径大小写**：Linux 服务器区分大小写，请确保图片路径大小写与实际文件一致
2. **图片格式**：推荐使用 PNG 或 JPEG 格式
3. **路径前缀**：所有图片路径以 `/images/` 开头（对应 `public/images/` 目录）
4. **ID 唯一性**：每个作品的 `id` 必须唯一，不能重复
